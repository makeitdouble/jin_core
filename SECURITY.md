# Security Policy

JIN Core is an experimental local-first project.

At this stage, there are no officially supported stable releases. The `master` branch may change quickly and should be treated as active development.

## Reporting a Vulnerability

If you find a security issue, please do not open a public issue with exploit details.

Send a report to:

`jincoreengine@gmail.com`

Please include:

* what the issue is
* how it can be reproduced
* what files or features are affected
* any relevant logs or screenshots

I will try to review security reports as soon as possible.

## Scope

Security reports are especially useful for issues related to:

* local file access
* browser/runtime actions
* prompt or memory injection
* unsafe handling of user data
* exposed API keys or credentials
* unintended network access

General bugs, setup problems, and feature requests should be opened as regular GitHub issues.
