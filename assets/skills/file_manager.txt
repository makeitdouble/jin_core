file_manager

Purpose:
Manage project asset files as external resources for operational tasks. Assets are files, not memory and not identity.

When to use:
- When the user asks to create, save, write, append, expand, update, preview, inspect, or store project asset files.
- When a loaded task-specific skill defines content rules, formats, folders, or naming constraints and the workflow also needs file operations.
- Use files instead of pasting large generated resources into chat.

Scope:
- Work only inside assets/.
- Use task-specific folders when another loaded instruction block defines them.
- Use assets/prompts/, assets/templates/, assets/outputs/, or another verified assets/ subfolder when appropriate.
- Never use absolute paths or parent traversal.

File behavior:
- Verify existing paths through the relevant project workflow before referencing or editing them.
- Create parent folders inside assets/ when useful.
- Do not delete files unless the user explicitly asks for deletion.
- Do not overwrite existing files unless the user explicitly asks to replace their contents.
- Prefer appending when the user asks to expand or add to an existing text resource.
- Preserve the target format required by the task-specific instructions.
- For plain line-based resources, one meaningful item per line is preferred unless the user asks for another structure.
- Keep generated lists in files; report only concise examples in chat.

Action workflow:
1. Use LIST_SKILLS before an operational file workflow if the relevant project skills are not already appended.
2. Use ASSET_ACTION with JSON payload for create_asset_file, append_asset_file, preview_file, and any task-specific asset action provided by another loaded instruction block.
3. Emit ASSET_ACTION as a JSON block:
<ASSET_ACTION>
{"action":"preview_file","path":"assets/outputs/example.txt","count":10}
</ASSET_ACTION>
4. Payload fields may be top-level or nested under args.
5. Use create_asset_file for new text files under assets/.
   Schema:
   {"action":"create_asset_file","path":"assets/outputs/example.txt","content":"line one\nline two"}
6. Use append_asset_file to add lines to an existing or new text file under assets/.
   Schema:
   {"action":"append_asset_file","path":"assets/outputs/example.txt","content":"line three"}
7. Use preview_file to inspect a small sample of an existing asset file.
8. After ASSET_ACTION completes, report paths, counts, and 3-5 examples when useful. Do not paste huge generated files into chat.
