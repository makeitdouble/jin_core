# Contributing to JIN Core

Thanks for checking out JIN Core.

This project is still experimental, so the most useful contributions right now are:

* bug reports
* setup issues
* clear examples where the runtime memory behaves wrong
* small fixes that make the project easier to run or understand

## Reporting bugs

When opening an issue, please include:

* your OS
* how you run the project
* the local model / provider you use
* what you expected to happen
* what actually happened
* logs, screenshots, or a short reproduction case if possible

A good bug report is more useful than a vague feature request.

## Suggesting features

Feature ideas are welcome, but please describe the actual use case first.

Good:

> I wanted JIN to remember X across the session, but it did Y instead.

Less useful:

> Add better memory.

## Pull requests

Small pull requests are easier to review.

Before opening a PR:

* keep the change focused
* avoid unrelated formatting changes
* explain what changed and why
* mention how you tested it

## Project direction

JIN Core is focused on runtime continuity for local LLMs: memory layers, visible process state, feedback signals, and a UI that helps the human and the model stay synchronized.
